Fibonacci数列四种求法

发表于 2020-04-11 | 分类于 数据结构
字数统计: 734 | 阅读时长 ≈ 3

题目

斐波那契数列指的是这样一个数列，1、1、2、3、5、8、13、21、34、……，这个数列从第3项开始，每一项都等于前两项之和。

解法

1.递归

public long fibA(int n) {//递归算法，但在在递归调用树中，存在大量的相同子问题计算 ,时间复杂度O(2^n)
    if(n==0) return 0;
    if(n==1) return 1;
    return fibA(n-1)+fibA(n-2);
}



如图所示，递归中存在相同的问题运算，使运算次数增加

2.递推

public long fibB(int n) {//递推算法 自底向上 ，子问题计算结果存放在表中,时间复杂度O(n) 
    long fibo[]= new long[100];
    if(n==0) return 0;
    if(n==1) return 1;
    fibo[0]=0;
    fibo[1]=1;
    for(int i=2;i<=n;i++)
       fibo[i]=fibo[i-2]+fibo[i-1];
    return fibo[n];    
}

递推：自底向上运算，借用了之前已经算出的值来进行下一次运算

3.优化递归

long fib[]= new long[100];
public long fibC(int n) {//自顶向下 ，保留递归调用，但使用已经计算的子问题的值，时间复杂度O(n)
    if(n==0) return 0;
    if(n==1) return 1;
    if(fib[n]!=0) return fib[n];
    return fib[n]=fibC(n-1)+fibC(n-2);
}

4.动态规划

public long fibD(int n) {//动态规划 当前值仅为前两次计算结果的总和, 时间复杂度O(n) 
    long front2=0,front1=1,sum = 0;
     for(int i=2;i<=n;i++)
     {
         sum=front2+front1;
         front2=front1;
         front1=sum;
    }
    return sum;
}

为了避免出现数太大溢出的问题，使用long 型，它的范围是-9223372036854775808 （-2的63次方）到9223372036854775807 （2的63次方-1）

}
